# Roadmap «Big Release 2026» — мастер-план до прод-деплоя

**Создан:** 2026-04-17
**HEAD на момент старта:** `558e1347`
**Прод отстаёт:** 273 коммита (деплой отложен до завершения всех 12 фаз)

## 🎯 Цель

Выйти на прод с согласованным SaaS-grade CRM: единый дизайн, нет функциональных конфликтов между разделами, работающие «Почта» и «Чат», ролевая аналитика, Android-приложение CRMProfiDialer, CI/CD, безопасность.

## 🗺️ Ключевые решения (зафиксированы после Q&A с пользователем 2026-04-17)

| Решение | Обоснование |
|---------|-------------|
| **Дизайн:** через скилл `frontend-design`, без Figma | Скорость итераций важнее pixel-perfect |
| **Mobile-версия: не нужна** | Все пользователи с ПК |
| **Dark mode: не актуально** | Экономия на 20% работы по стилям |
| **Layout:** «современные CRM» (HubSpot/Salesforce/Linear-style) + Notion-типографика | Баланс функциональности и визуала |
| **«Live-chat» → «Чат»** | Переименовать везде в UI |
| **Внутренний чат (Slack-стиль): не нужен** | Только клиент-операторная коммуникация |
| **Почта:** только массовые рассылки | Без IMAP/SMTP per-user, без входящей |
| **Аналитика:** свободная разработка, понятная, с графиками | Claude определяет KPI по ролям |
| **Android:** `CRMProfiDialer` (Kotlin + Room), внутренний APK | Dialer-функция: клик→звонок. Без записи |
| **Роли (6):** Менеджер, Тендерист, РОП, Директор подразделения, Управляющий группой компаний, Администратор | Соответствие к коду: MANAGER, TENDERIST, SALES_HEAD, BRANCH_DIRECTOR, GROUP_MANAGER, ADMIN. Уточнено 2026-04-17. |
| **CI/CD:** GitHub Actions (авто-staging, ручной прод) | Единый индустриальный подход |
| **Blue/green deploy:** пока не нужен | 5-сек downtime приемлем |
| **Работа:** автономная, связь только в критических ситуациях | User доверяет |

## 📋 Правила работы (применять автоматически)

1. **В начале каждой фазы** — скилл `writing-plans` + записать стоп-точку в `current-sprint.md`
2. **После каждого коммита** — обновить `docs/current-sprint.md`
3. **Каждая решённая проблема** → `docs/problems-solved.md` (формат: Симптом / Причина / Решение)
4. **Каждое архитектурное решение** → `docs/decisions.md` (ADR)
5. **После крупного изменения моделей/кода** → `/graphify --update`
6. **Перед коммитом** — скилл `verification-before-completion` (Django check + тесты + ruff)
7. **После всей фазы** — `/health-bugs` + `/health-security` + staging smoke
8. **Мелкие идеи/баги на потом** → spawn-task чип или `docs/roadmap.md` секция «Backlog»
9. **Критические развилки** → связаться с user (не принимать в одиночку решение с невосстановимыми последствиями: удаление данных, смена контракта API, смена ролей)
