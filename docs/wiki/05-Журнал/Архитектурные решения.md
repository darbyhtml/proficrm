## ADR-001: gthread вместо gevent для Gunicorn

**Дата:** 2026-04-06
**Контекст:** SSE-стримы блокировали sync workers (2 шт). Нужен concurrency.
**Варианты:**
1. gevent (greenlets) — конфликт с psycopg3 (`Queue[T]` не subscriptable)
2. gthread (реальные потоки) — совместим со всем стеком
3. Увеличить sync workers до 8+ — неэффективно

**Решение:** gthread (4 workers × 8 threads = 32 соединения)
**Почему:** psycopg3 несовместим с gevent monkey-patching.

## ADR-002: CORS для виджета — разделение nginx/Django

**Дата:** 2026-04-05
**Контекст:** Браузер получал дублирующиеся CORS-заголовки (nginx + Django).
**Решение:**
- Nginx: обработка OPTIONS preflight для `/api/widget/`
- Django: добавление CORS на ответы через `_add_widget_cors_headers()`
- django-cors-headers: только для основного API

**Почему:** django-cors-headers middleware перехватывает OPTIONS до view-кода. Поэтому custom handler в Django не работает для preflight. Nginx обрабатывает preflight до проксирования.

## ADR-003: SSE вместо WebSocket для real-time

**Дата:** 2026-04-02
**Контекст:** Виджет на внешнем сайте. WebSocket требует отдельный origin handling.
**Решение:** SSE (Server-Sent Events) с короткими соединениями (25-55с).
**Почему:**
- SSE работает через обычный HTTP (проще CORS)
- EventSource API автоматически reconnect'ит
- Fallback на polling встроен
- Для one-directional push (сервер → клиент) SSE достаточно

## ADR-004: PostgreSQL FTS вместо Elasticsearch/Typesense

**Дата:** 2026-03 (ранее)
**Контекст:** Нужен поиск компаний (нечёткий + полнотекстовый).
**Решение:** PostgreSQL FTS + pg_trgm.
**Почему:**
- Нет лишних зависимостей
- Достаточная производительность для текущего масштаба
- tsvector/tsquery + similarity покрывают требования
- Переиндексация: 1 Celery-задача в день

## ADR-005: Fernet для шифрования SMTP-паролей

**Дата:** 2026-03 (ранее)
**Контекст:** SMTP-пароли хранятся в БД.
**Решение:** Fernet (symmetric, `cryptography` library).
**Почему:** Простой, стандартный, обратимый (нужен для отправки).
