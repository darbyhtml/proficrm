# База данных

PostgreSQL 16, 55 моделей.

## Модели по приложениям

### accounts (3 модели)
- `Branch` — филиалы/офисы
- `User` — пользователи с ролями и data scope
- `MagicLinkToken` — одноразовые токены входа (SHA256, TTL 24ч)

### companies (16 моделей) — ядро CRM
- `Company` — компания (менеджер, контракт, кастомные поля)
- `Contact` — контактные лица
- `CompanyNote` / `CompanyNoteAttachment` — заметки с вложениями
- `CompanyDeal` — сделки
- `CompanyEmail` / `CompanyPhone` — контактные данные компании
- `ContactEmail` / `ContactPhone` — контакты физ. лиц
- `CompanyStatus` / `CompanySphere` / `Region` / `ContractType` — справочники
- `CompanyDeletionRequest` — soft-delete с подтверждением
- `CompanySearchIndex` — полнотекстовый индекс (FTS + trgm)
- `CompanyHistoryEvent` — аудит изменений

### messenger (16 моделей) — live-chat
- `Inbox` — входящие каналы (widget token, branch)
- `Channel` — типы каналов (website, Telegram, WhatsApp, VK, Email)
- `Contact` — контакты мессенджера (отдельная от companies)
- `Conversation` — диалоги (статусы: new→assigned→waiting→resolved→closed)
- `Message` / `MessageAttachment` — сообщения и вложения
- `ContactInbox` — привязка контакт↔inbox
- `RoutingRule` — маршрутизация (GeoIP, device)
- `CannedResponse` — шаблоны быстрых ответов
- `ConversationLabel` — метки
- `AgentProfile` — профили операторов
- `PushSubscription` — push-подписки
- `Campaign` — кампании мессенджера
- `AutomationRule` — автоматизация (приветствие, эскалация)
- `ReportingEvent` — метрики
- `Macro` — горячие клавиши операторов

### mailer (10 моделей) — рассылки
- `MailAccount` — SMTP-аккаунт пользователя
- `GlobalMailAccount` — глобальный SMTP (smtp.bz, Fernet)
- `Campaign` / `CampaignRecipient` — кампании и получатели
- `CampaignQueue` — очередь отправки (приоритет, deferrral)
- `SendLog` — логи отправки (Message-ID, идемпотентность)
- `Unsubscribe` / `UnsubscribeToken` — отписки
- `EmailCooldown` — cooldown после отписки
- `SmtpBzQuota` — квоты smtp.bz

### tasksapp (4 модели)
- `TaskType` — типы задач (иконки, цвета)
- `Task` — задачи (UUID PK, RRULE повторения)
- `TaskComment` — комментарии
- `TaskEvent` — история изменений

### notifications (4 модели)
- `Notification` — уведомления (типизированные)
- `CompanyContractReminder` — напоминания по контрактам
- `CrmAnnouncement` / `CrmAnnouncementRead` — системные анонсы

### phonebridge (6 моделей)
- `PhoneDevice` — устройства (FCM token)
- `CallRequest` — входящие/исходящие вызовы
- `PhoneTelemetry` — телеметрия устройств
- `PhoneLogBundle` — пакетные логи вызовов
- `MobileAppBuild` — версии Android-приложения
- `MobileAppQrToken` — QR-спаривание

### audit (2 модели)
- `ActivityEvent` — действия пользователей (retention 180 дней)
- `ErrorLog` — ошибки системы (retention 90 дней)

### policy (2 модели)
- `PolicyConfig` / `PolicyRule` — конфигурация политик

## Ретенция данных

| Данные | Срок хранения | Задача очистки |
|--------|-------------|----------------|
| ActivityEvent | 180 дней | Еженедельно, Вс 03:00 |
| ErrorLog | 90 дней | Еженедельно, Вс 03:15 |
| Notification | 90 дней | Еженедельно, Вс 03:30 |
