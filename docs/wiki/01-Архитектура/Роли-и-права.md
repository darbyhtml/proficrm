# Роли и права доступа CRM ПРОФИ

Полная матрица прав на 2026-04-19. Источник истины: `backend/accounts/models.py` (User.Role), `backend/companies/permissions.py`, `backend/ui/views/_base.py`, `backend/policy/engine.py`.

## 6 ролей системы

| UI (русский) | Code (`User.Role`) | Scope | Главное назначение |
|---|---|---|---|
| **Менеджер** | `MANAGER` | свои компании + задачи | ведёт клиентов, отвечает в чате |
| **Тендерист** | `TENDERIST` | read-only + свои заметки | просмотр без задач, без CRUD компаний |
| **РОП** | `SALES_HEAD` | своё подразделение | управление менеджерами, эскалации |
| **Директор подразделения** | `BRANCH_DIRECTOR` | своё подразделение + transfer | управление филиалом |
| **Управляющий группой** | `GROUP_MANAGER` | все подразделения | executive, все операции (кроме sensitive) |
| **Администратор** | `ADMIN` | полный доступ + /settings/ | техадмин, reset cold_call, экспорт CSV |

> **Важно**: `GROUP_MANAGER ≠ ADMIN`. GROUP_MANAGER — executive-наблюдатель без `/settings/`; ADMIN — техадмин с `/admin/` и системными настройками.
>
> **TENDERIST** (добавлен 2026-04-17) — read-only роль, исключена из live-chat, не может создавать задачи/компании.

### Функции scope

```python
# backend/ui/views/_base.py

def _can_edit_company(user, company):
    if user.is_superuser or user.role == ADMIN: return True
    if user.role == GROUP_MANAGER: return True
    if user.role in (SALES_HEAD, BRANCH_DIRECTOR):
        return company.branch_id == user.branch_id
    if user.role == MANAGER:
        return company.responsible_id == user.id
    return False  # TENDERIST и все остальные

def _can_delete_company(user, company):
    # Как edit + MANAGER не может удалять direct (только request)
    return _can_edit_company(user, company) and user.role != MANAGER
```

```python
# backend/accounts/permissions.py
def require_admin(user: User) -> bool:
    return bool(user and user.is_authenticated and user.is_active
                and (user.is_superuser or user.role == User.Role.ADMIN))
```

## View As (режим «смотреть как другой пользователь»)

| Функция | Доступно для |
|---------|---|
| Включение режима | ADMIN (+ superuser) |
| Выбор пользователя | ADMIN |
| Действия под выбранным user | Как если бы вы были тем пользователем |

```python
# backend/crm/utils.py
def get_effective_user(request) -> User:
    view_as_user = get_view_as_user(request)  # требует is_superuser or role==ADMIN
    return view_as_user or request.user
```

Сессионные ключи: `view_as_enabled` (bool), `view_as_user_id` (int), `view_as_role` (str), `view_as_branch_id` (int).

## Удаление компании: Flow запросов

```
MANAGER (свои) / SH / BD (филиал)                    ADMIN / GROUP_MANAGER
         │                                                   │
         ▼                                                   ▼
   «Запросить удаление»                             «Рассмотреть»
         │                                                   │
         ▼                                                   ▼
  CompanyDeletionRequest                   approve │        │ cancel
     (status=pending)                              ▼        ▼
                                            Удалить         Отклонить
                                            (CASCADE        (уведомить
                                             tasks + co)     автора)
```

Ресурсы политики:
- `ui:companies:delete_request:create` (sensitive)
- `ui:companies:delete_request:approve` (sensitive)
- `ui:companies:delete_request:cancel` (sensitive)

**CASCADE delete** (с 2026-04-19, коммит `b7dcb21a`): при удалении компании — сначала `Task.objects.filter(company_id=company_pk).delete()`, затем `company.delete()`. Количество удалённых задач пишется в `ActivityEvent.meta["tasks_deleted_count"]`.

## Фильтр «Ответственный» в списке компаний/задач

Функция `companies.permissions.get_users_for_lists(user, exclude_admin=True)`:
- возвращает **ВСЕ активные** non-TENDERIST, non-ADMIN пользователи (MANAGER/SALES_HEAD/BRANCH_DIRECTOR/GROUP_MANAGER)
- сортировка: по branch → last_name → first_name
- **не режется** по текущему user — РОП видит всех для операций передачи

## Data Scope в модели User

```python
# backend/accounts/models.py
class DataScope(models.TextChoices):
    GLOBAL = "global"   # (по умолчанию) вся база
    BRANCH = "branch"   # только своё подразделение
    SELF = "self"       # только свои компании (responsible_id)
```

В Web UI все видят «всю базу» (GLOBAL). Ограничения на действия применяются через `@policy_required()` + функции `_can_*` выше.
