# Дизайн-система v3

> **Статус:** F1 Big Release 2026 (2026-04-17)
> **Основание:** v2 (`backend/templates/ui/_v2/v2_styles.html`) + HubSpot/Linear best practices
> **Цель:** единый визуальный язык для всех 9 разделов перед прод-деплоем

## Принципы

1. **Notion-типографика.** Плотно, но не мелко. Базовый текст 15px, вторичный 13px, мета 11px.
2. **Спокойные переходы.** 150ms ease. Никаких bounce/spring. Уважаем `prefers-reduced-motion`.
3. **Один визуальный уровень элевации.** Карточки с лёгкой тенью. Модалки — сильная тень. Никаких shadow-войн.
4. **Семантические цвета.** Не «оранжевый», а «accent». Не «красный», а «danger». Меняется в одном месте.
5. **Accessibility by default.** Контраст AA (4.5:1) минимум. Focus-visible всегда. ARIA на интерактиве.
6. **Desktop only.** Mobile не поддерживаем (решение user, Q2). Min-width 1280px.
7. **Без dark mode.** Экономим 20% работы по стилям (решение user, Q3).
8. **CSP-strict ready.** Ни одного inline `onclick`/`onchange`/`onfocus`. Всё через addEventListener.

## Компоненты

### 1. Button `.v3-btn`

**Варианты:**
- `.v3-btn` — primary (teal fill, white text)
- `.v3-btn--ghost` — outlined (transparent, border, text)
- `.v3-btn--danger` — red fill
- `.v3-btn--sm` — компактная (height 30px)
- `.v3-btn--lg` — крупная (height 44px для CTA)

**Размеры:**
- `sm`: 30px height, 13px text, 10px padding
- `default`: 36px height, 14px text, 14px padding
- `lg`: 44px height, 15px text, 20px padding

**States:** hover, active (`:active scale(0.98)`), disabled (opacity 0.5, cursor not-allowed), focus-visible (ring).

### 2. Input `.v3-input`, Select `.v3-select`, Textarea `.v3-textarea`

Высота 36px (sm — 30px, lg — 44px). Border radius 6px. Focus-ring teal.

**Валидация:** `.v3-input--error` (red border + error-text ниже), `.v3-input--success` (green border).

### 3. Card `.v3-card`

```
.v3-card
├── .v3-card__head
│   ├── .v3-card__title (h2)
│   └── .v3-card__actions
├── .v3-card__body
└── .v3-card__foot
```

**Варианты:** `--hover`, `--flat` (no shadow), `--interactive` (cursor pointer).

### 4. Badge / Chip / Count

- `.v3-badge` — inline ярлык (статус/метка)
- `.v3-chip` — интерактивный фильтр/тег
- `.v3-count` — числовой индикатор

Цвета через модификаторы: `--primary`, `--accent`, `--danger`, `--warn`, `--success`, `--info`, `--neutral`.

### 5. Table (grid-based, не `<table>`)

```
.v3-table
├── .v3-tr.v3-tr--head
└── .v3-tr
    └── .v3-td
```

`grid-template-columns` задаётся inline-style (единственное допустимое inline для grid).
Hover строки: `background: var(--v3-bg-soft)`. Overdue: `--v3-tr--overdue` с левой danger-полосой.

### 6. Modal `.v3-modal`

**Структура:**
```
.v3-modal (role="dialog" aria-modal="true" aria-labelledby="id")
├── .v3-modal__backdrop (click → close)
└── .v3-modal__dialog
    ├── .v3-modal__head
    │   ├── h2#id (title)
    │   └── button[aria-label="Закрыть"]
    ├── .v3-modal__body
    └── .v3-modal__foot (actions)
```

**Обязательно:** focus trap, Escape key, возврат фокуса при close, `aria-hidden="true"` при закрытии.

### 7. Toast `.v3-toast`

Позиция: bottom-right. Auto-dismiss 4 сек (error — 6 сек). `role="status"` / `role="alert"`.
Варианты: default, success, error, warn, info.

### 8. Skip-link

Обязательный элемент базового шаблона. `href="#main-content"`, visible on focus.

### 9. Sort header

```html
<button class="v3-sort" aria-sort="ascending|descending|none">
    Название <svg .../>
</button>
```

Стрелка ▲/▼ визуально + `aria-sort` для screen reader.

### 10. Filter popover

```html
<div class="v3-filter">
    <button class="v3-btn v3-btn--ghost" aria-haspopup="true" aria-expanded="false">
        Фильтр <span class="v3-count v3-count--primary">3</span>
    </button>
    <div class="v3-filter__popover" role="dialog" aria-label="Фильтры">
        ...
    </div>
</div>
```

Focus trap внутри popover. Закрытие по Escape и клику снаружи.

## Паттерны

### Empty state

```
┌─────────────────────────────────┐
│         [icon, muted]           │
│     «Задач пока нет»            │
│  Создайте первую для старта     │
│    [+ Создать задачу] (CTA)     │
└─────────────────────────────────┘
```

### Confirmation

**Запрещено:** `window.confirm()` / `alert()` — не работают на touch, не доступны screen reader, блокируют JS.

**Стандарт:**
- Для неразрушительных действий: toast с undo (5 сек)
- Для разрушительных (delete): модалка с явным текстом + двойным подтверждением
- Для быстрых (status change): двойной клик с visual badge «Нажмите ещё раз» (2.5 сек)

### Loading / Skeleton

Никаких спиннеров во весь экран. Точечные:
- `.v3-spinner` — inline 16px (внутри кнопки)
- `.v3-skeleton` — серый плейсхолдер (shimmer optional)

### Indicator (poll / websocket)

Маленький fixed в top-right, `role="status"` + `aria-live="polite"`.

## Accessibility чеклист (для каждой новой страницы)

- [ ] `<main id="main-content" tabindex="-1">` — для skip-link
- [ ] Все кнопки имеют текст **или** `aria-label`
- [ ] Декоративные SVG имеют `aria-hidden="true"` + `focusable="false"`
- [ ] Информативные SVG имеют `role="img"` + `aria-label`
- [ ] Формы: `<label>` привязаны к input через `for="id"`
- [ ] Required поля помечены `required` + визуально
- [ ] Ошибки формы через `aria-describedby` + `role="alert"`
- [ ] Табличные sort-headers имеют `aria-sort`
- [ ] Модалки: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`, focus trap
- [ ] Контраст 4.5:1 minimum (3:1 для крупного текста 18px+)
- [ ] Keyboard-only полный flow (no mouse traps)
- [ ] `:focus-visible` ring на всех интерактивных элементах

## Performance правила

- Каждая страница не должна делать >10 SQL-запросов на render. N+1 = блокер.
- `.only(...)` + `select_related` по необходимости.
- Тяжёлые агрегаты (`.count()`) — только 1 раз на страницу, лучше через `fetch [:limit+1] + len`.
- Polling: не чаще 30 сек + visibility pause + exponential backoff + jitter.
- Inline-критический CSS в `extra_head` — только для выше-скролла. Остальное — в `main.css`.

## Как мигрировать v2 → v3

1. **Не ломаем старое.** `v2_styles.html` остаётся, добавляем `v3_styles.html` рядом.
2. Токены `--v2-*` остаются алиасами на `--v3-*` для обратной совместимости.
3. Новые компоненты (`.v3-*`) — только в новых шаблонах или при рефакторинге старых.
4. Каждая фаза F3-F8 рефакторит свою страницу на v3-токены.
5. После F8 — удаляем v2-алиасы (только `--v3-*` остаются).
