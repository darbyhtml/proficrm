## 7. Критические пути

### 1. Создание компании
```
POST /api/companies/
  → CompanyViewSet.create()
  → Company.objects.create()
  → @post_save signal
  → schedule_rebuild_index (async)
  → rebuild_company_search_index → FTS update
```

### 2. Live-chat + auto-assign
```
POST /api/widget/send-message
  → Conversation.create() или attach
  → auto_assign_conversation (round-robin per-branch)
  → User.get_online_operators() → Redis cache
  → Message created
  → @post_save signal → WebSocket notify (channels_redis)
  → WS /ws/operator/ ← notify_new_message
```

### 3. Email Campaign
```
POST /api/mailer/campaigns/{id}/start
  → Campaign.start() → CampaignQueue.create(status=PENDING)
  → celery-beat: send_pending_emails (60s)
  → decrypt SMTP password (Fernet)
  → smtplib → external SMTP.BZ
  → update CampaignRecipient.status
```

### 4. Удаление компании (CASCADE hotfix)
```
POST /companies/{id}/delete/
  → company_delete_direct()
  → Task.objects.filter(company_id).delete()  ← Hotfix 2026-04-18
  → Company.delete()
  → CASCADE: Contact, CompanyNote, Phone, Email, Deal, ...
  → audit.log_event(meta: {tasks_deleted_count: N})
```

## 10. Ключевые insights

1. **User** — суперузел (747 edges): глобальная авторизация, все FK сходят к User.
2. **Redis** — критическая инфра: broker, cache, Channels layer, online status, rate limit.
3. **Company** — центр сущностей: 16 child-моделей с CASCADE (+ Task через manual CASCADE в view).
4. **Асинхронность**: 3 Celery сервиса (worker, beat, result backend).
5. **RRULE** — сложная логика: parsing, generation на 30 дней, Redis lock от race.
6. **Мессенджер** — WebSocket-first через Daphne; auto-assign через round-robin per-branch; Redis online status.
7. **Почта** — external SMTP.BZ, Fernet encryption, 60-second batch, quota sync.
