## Findings per workflow

### Workflow 1: Manager's Morning (dashboard `/`)

**Observations**: Full dashboard aggregating tasks + contracts + companies.

- Heading: "Доброе утро, <username>".
- 4 summary tiles (clickable, linked to filtered `/tasks/?mine=1&...`):
  - На сегодня (today).
  - Просрочено (overdue).
  - На неделю (next 7 days).
  - Готово сегодня (done today).
- 5 expanded cards ниже:
  - Просрочено (list).
  - На сегодня (+ "Создать задачу" inline button).
  - Новые задачи.
  - Договоры (contract reminders).
  - Ближайшие 7 дней.
  - Компании без активных задач.
- Top-right quick-add: **+ Задача** (primary CTA), Обновить, "Отчёт: день/месяц".
- **Clicks to today's tasks**: 0 (visible on dashboard) — excellent.

**Pain points**:
- No "recent activity" (чат/звонки/emails) — activity focus is **tasks only**.
- No "priority clients" concept.
- No "what's new" changelog per company.

### Workflow 2: Company deep-dive (`/companies/<id>/`)

**Observations**: **Very strong single-hub pattern** — amoCRM paradigm #2 delivered.

All sections in one scrollable page:
- Header: breadcrumb + title + ИНН/КПП/Область inline edit + quick actions
  (Редактировать данные, Удалить, tasks count, attachments, search icon).
- **Данные** card: 12+ fields (Юр. название, Сайт, Вид деятельности,
  Численность, Часовой пояс, Режим работы, Руководитель, Договор, Телефон,
  Комментарий, Email, Контакт ФИО, Контакт должность, Адрес). Every field
  has **inline pencil edit** ✅.
- **Быстро** card: **+ Поставить задачу** (primary CTA) + "Передать другому
  менеджеру" dropdown (transfer workflow) + Статус + Сферы.
- **Контакты** card (+ Добавить контакт).
- **Последние задачи** card (+ Добавить задачу, История link).
- **Заметки** card: inline textarea with send button, chronological notes
  с edit/pin/delete per note.
- **История действий** (collapsed): shows `Добавлена заметка` + `Статус: X`
  events — но **не calls/emails/chat**.

**Clicks to see all company info**: 1 (navigate from list) → everything
visible на одной странице. amoCRM ~equivalent.

### Workflow 3: Task creation с context

**Observations**: From company detail → click "+ Поставить задачу":

- **Modal opens** (не переход на отдельную страницу) ✅.
- Fields: Задача (type dropdown), Описание, **Компания** (pre-filled =
  current company ✅), Кому (assignee), Дедлайн, Срочная задача checkbox.
- "Менеджер может назначать только себе" policy hint.

**Clicks from company view to saved task**: 1 (open modal) + fields + Создать = ~2-3 clicks total. Excellent.

### Workflow 4: Communications timeline (`/messenger/`)

**Observations**: Chat = **separate page**, not integrated в company card.

- Left: dialogs list + status filter + "Мои" toggle.
- Right: conversation pane (empty если не выбран диалог).
- "Нажмите Ctrl+K для быстрого поиска" hint.
- Currently no dialogs existing в staging.

**Gap**: Chat/email/call activity per company visible **only через admin
pages** (admin/calls/stats/) или требуют navigation из messenger по
идентификатору клиента. Company detail "История действий" — только
notes + status changes. **No unified chronological timeline**.

### Workflow 5: Global search (Ctrl+K)

**Observations**:
- Ctrl+K focuses search input of current page (e.g. messenger dialog search,
  companies list phone/email search).
- **NO global cross-entity search** (find "Ivanov" → companies + contacts +
  tasks + deals).
- Each page has own scoped search box.

**Gap vs amoCRM**: CRITICAL. amoCRM has persistent global search в top bar
на любой странице.

### Workflow 6: Pipeline view / Sales process

**Observations**: Navigated к `/analytics/` — aggregated activity view:

- Grouped by подразделение (ЕКБ/Краснодар/Тюмень) + "Без подразделения".
- Per-manager tiles: Звонков / Холодных count, "Открыть" link.
- Period: Day / Month toggle.

**Gap**: **NO sales pipeline / kanban view exists**.

- `CompanyDeal` model в backend — stub только (program, price_per_person,
  listeners_count, no status/stage/probability).
- 0 deals в prod (feature unused).
- Status на `Company` — ForeignKey к `CompanyStatus` dictionary, не sales
  stage flow.
- No drag-drop stages visualization.

amoCRM's core feature (Leads pipeline) **missing entirely**.

### Workflow 7: Bulk operations

**Observations** (company list `/companies/`):

- **No row-level checkboxes** ❌.
- Filters rich (Статус / Сфера / Регион / Подразделение / Ответственный
  multi-select), "Применить" to filter.
- Actions on list: Экспорт CSV + Новая компания (bulk-agnostic).

**Observations** (tasks list `/tasks/`):

- **Row checkboxes present** ✅.
- Top bulk bar when select > 0: **ПЕРЕНАЗНАЧИТЬ** (assignee dropdown)
  + **ПЕРЕНЕСТИ СРОК** (date shift).
- "Выбрано: 0" counter.

**Inconsistency**: Bulk exists для tasks, missing для companies —
несмотря на то что backend уже имеет `company_bulk_transfer_preview` +
`company_bulk_transfer` views (`ui/views/company_list.py:557, 713`).
UI wiring просто не done.

### Workflow 8: After-call workflow

**Observations**: Call events logged из Android app (ActivityEvent +
phonebridge CallRequest), displayed на web только через admin aggregates:

- `/admin/calls/stats/` — per-manager aggregated table (Всего / Дозвонился
  / Не дозвонился / Занято / Отклонен). Admin-only, not per-company.
- Company detail "История действий" — только notes + status; не calls.
- **No per-company call timeline** on web.
- No "create follow-up task from call" flow.

**Gap**: Call data существует в backend (CallRequest + phonebridge),
но not surfaced в company UI. Manager не может на web видеть "я звонил
этому клиенту вчера 3 раза".

## Top 5 recommended quick wins

Ordered by impact/effort ratio.

### #1: Bulk select для companies list

- **Current flow**: To transfer 10 companies к new manager → 10× open card,
  10× select dropdown, 10× save. ~30 clicks + 10 page transitions.
- **Proposed flow**: Checkbox on rows + "Переназначить" dropdown (same
  pattern as tasks) → 12 clicks (10 select + 1 dropdown + 1 apply).
- **User impact**: 10× companies transfer = ~18 clicks saved + 10 pages
  avoided.
- **Implementation effort**: **Small** (~2-3h). Backend views already
  exist (`company_bulk_transfer_preview` + `company_bulk_transfer`).
  Work: add checkboxes + "Выбрано: N" counter + bulk bar к company list template.
- **Risk**: LOW (reuses proved task bulk pattern).

### #2: Global search Ctrl+K (cross-entity)

- **Current flow**: To find "Иванов" — navigate к Companies list, type,
  then to Contacts list, type again, then Tasks etc. 3-4 separate
  searches.
- **Proposed flow**: Single Ctrl+K modal anywhere → results grouped
  Companies / Contacts / Tasks / Notes / Deals.
- **User impact**: ~2-5s saved per search, но daily × 10-20 uses = major
  cognitive+time win.
- **Implementation effort**: **Medium** (~4-6h). New endpoint
  `/api/search/?q=`, fan-out к FTS на Company + Contact + Task + Note
  tables (CompanySearchIndex already exists). Modal UI component.
- **Risk**: Low-Medium. FTS performance на 45K companies + 99K contacts —
  test с EXPLAIN.

### #3: Unified timeline on company detail

- **Current flow**: To see "what happened с этой компанией" — read notes
  (one card), check задачи (another card), look admin/calls/stats (admin-
  only, aggregated), check messenger separately (navigate away).
- **Proposed flow**: Expand "История действий" к include all events for
  `company_id=X`: calls, notes, tasks created/completed, status changes,
  emails sent, transfer events. ActivityEvent model already contains
  most.
- **User impact**: "Полная картина клиента за 5 секунд" — replaces 4+
  navigation paths.
- **Implementation effort**: **Medium** (~4-8h). Backend: query
  ActivityEvent для company с N types filter. Frontend: iconified
  timeline (phone/note/task/email icons) в existing `<details>` section.
- **Risk**: Medium. Need pagination (prod companies могут иметь 500+
  events). Use composite index `(entity_type, created_at DESC)` added в W3.2.

### #4: Quick-add menu (floating + or top bar)

- **Current flow**: To add note/contact/task from non-company page — navigate
  к company first. Create flow тяжёлая если в середине другой задачи.
- **Proposed flow**: Header "+ New" dropdown (amoCRM pattern) с вариантами:
  Задача, Компания, Контакт, Заметка. Pre-opens modal.
- **User impact**: 2-3 clicks saved per quick-create. High daily frequency.
- **Implementation effort**: **Small** (~2h). Add header dropdown button +
  4 modal wiring (already exists for Task; Company via /companies/new/).
- **Risk**: LOW.

### #5: Call entries в company history

- **Current flow**: Call events logged в ActivityEvent + phonebridge but
  display — admin aggregated stats only. Manager не видит "был ли звонок"
  в card.
- **Proposed flow**: Add `phone_call` entity_type events к company
  "История действий" с info: direction (in/out), duration, timestamp,
  outcome (answered/missed).
- **User impact**: Eliminates blind spot — manager видит history of
  attempts per client.
- **Implementation effort**: **Small-Medium** (~3-4h). Data already
  collected by Android CallListenerService. Work: ensure company
  linkage (phone → company lookup exists), render в history section.
- **Risk**: Medium (phone-to-company matching accuracy + permissions —
  call count не должен raise privacy concerns).

**Total Top-5 effort**: ~15-25h distributed. All low-medium risk.
After delivery, 4 of 8 amoCRM patterns improve; remaining gaps =
pipeline kanban + calendar view (deferred к W9).

## Deferred к W9 (full UX wave)

Large-scope changes out of quick-win scope:

1. **Sales pipeline (kanban)** — requires: rewrite `CompanyDeal` с stages
  + probability, drag-drop UI component, stage transitions audit,
  conversion analytics. **~30-50h**. Coordinate с product decisions
  (does GroupProfi actually need leads pipeline? 0 deals currently used).

2. **Calendar view for tasks** — day/week/month grid с drag-to-reschedule.
  ~20h.

3. **Email threading in timeline** — requires SMTP+IMAP integration
  (currently SMTP-only for campaigns, no inbound parsing).

4. **Priority clients concept** — "My key accounts" filter + dashboard
  tile + sticky sort. ~6h.

5. **Dashboard customization** — hide/show cards per user preference
  (analytics widget, contracts, etc).

6. **Task types с custom icons** — already have per-type badges (lines
  Перезвонить / Связаться / Обсудить КП), extend to dashboard aggregation.

7. **Company card v3/b full rollout** — per `project_f4_r3_v3b_companies.md`
  memory, already implemented but awaiting user approval to replace classic.

## Verdict

**GroupProfi UX — 4/8 amoCRM patterns well-delivered** (single-hub company
card, inline editing, quick-add tasks, rich filters). **3/8 critical gaps**:
pipeline kanban, unified timeline, global search. Tagline "простая, как
amo, но без лишнего" accurate для paradigms 2, 5; misleading для 1, 6, 7.

**Recommended next step**: **execute Top-5 quick wins** (~15-25h across 2-3
sessions) prior к larger W9 UX wave. All 5 have backend support already;
work is mainly UI wiring.
