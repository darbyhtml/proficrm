# Шкала z-index в UI ProfiCRM

Чтобы избежать наложения элементов (например, панель контакта поверх bulk-действий задач), в проекте используется единая шкала z-index.

| Значение | Назначение | Примеры |
|----------|------------|---------|
| **30** | Sticky-хедер (фиксированная шапка) | Шапка карточки компании (Modern) |
| **50** | Всплывающие окна и выпадающие меню верхнего уровня | Модалка «Добавить email в кампанию», меню быстрых действий компании (Modern) |
| **60** | Dropdown-меню внутри карточек (копировать/изменить) | Все `data-dd-menu` в карточке компании |
| **100** | (зарезервировано) | — |
| **120** | Немодальная панель поверх страницы | Панель создания/редактирования контакта (`#contactPanel`) |
| **200** | Модальные окна | Удаление компании, договор, заметки, задачи (создание/редактирование/удаление/выполнение) |
| **220** | Модалки поверх других модалок | Просмотр изображения (превью) |

## Правила

- Не использовать произвольные значения (например, 999). Добавлять новые уровни только из этой шкалы.
- Панель контакта (120) выше dropdown (60), но ниже полноэкранных модалок (200), чтобы при открытой модалке задачи панель контакта не перекрывала её.
- При добавлении нового слоя (например, глобальный тултип) — завести новый уровень и обновить этот документ.
